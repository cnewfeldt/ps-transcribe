# Roadmap: PS Transcribe v1.1 — Marketing Website

**Status:** ✅ SHIPPED 2026-04-25
**Phases:** 11–14
**Total Plans:** 16

## Overview

Four phases delivered a marketing website for PS Transcribe at `ps-transcribe-web.vercel.app`: scaffold a Next.js project inside the existing repo and connect Vercel deployment first (so every later change previews on a URL), port the Chronicle design tokens and primitives second (so every later page inherits the visual system), then build the two content surfaces — landing, then docs — in dependency order.

The user fed the Claude Design brief (`.planning/research/CLAUDE-DESIGN-BRIEF.md`, BRIEF-01 complete) into Claude Design to produce HTML/CSS mocks, which served as the visual input for phases 13 and 14.

**2026-04-25 scope reduction:** Phase 15 (Changelog Page) was scoped, planned, and built but reverted. Public release notes are out of scope for v1.1; `CHANGELOG.md` stays a developer-internal artifact (no `/changelog` route, no RSS feed, no nav link).

**Slug fallback:** `ps-transcribe.vercel.app` was already claimed on Vercel by another account at phase 11 setup time, so the fallback slug `ps-transcribe-web.vercel.app` is the production host. A custom domain (v1.2 candidate) will supersede this.

### Phase 11: Website Scaffolding & Vercel Deployment
**Goal**: A `/website` subdirectory exists with a working Next.js App Router + TypeScript project, deploys on every push, produces preview URLs per PR, and serves production at `ps-transcribe-web.vercel.app` — without polluting the Swift package or committing build artifacts.
**Depends on**: Nothing (first v1.1 phase; v1.0 complete)
**Requirements**: SITE-01, SITE-02, SITE-03, SITE-04, SITE-05
**Success Criteria** (what must be TRUE):
  1. Running `npm install && npm run dev` inside `/website` boots a Next.js App Router dev server on localhost with TypeScript compiling cleanly
  2. Pushing any commit that touches `/website` to a branch produces a Vercel preview URL visible on the PR
  3. `ps-transcribe-web.vercel.app` serves the latest `main`-branch build (blank-but-live is acceptable at this phase)
  4. `git status` shows no `.next/` or `node_modules/` files ever staged from the `/website` subdir
  5. The Swift package at the repo root still builds; the website has its own isolated `package.json`
**Plans:** 3/3 plans complete

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

- v1.1 phase numbering continues v1.0 — first v1.1 phase is 11, not 1 (rationale: phase numbers identify execution order across the project lifetime, not within a milestone)
- Website lives in `/website` subdirectory of the existing repo (rationale: avoid spinning up a second repo when the website's lifecycle is tightly coupled to the macOS app's release cadence)
- Chronicle design system ported from the macOS app, not redesigned for the web (rationale: visual cohesion between app and marketing surface; light mode only because the macOS app is light-only)
- Vercel subdomain only in v1.1; custom domain deferred (rationale: ship the site first, decide on domain in v1.2 once content has settled)
- Production slug fallback to `ps-transcribe-web.vercel.app` (rationale: the canonical `ps-transcribe.vercel.app` was already claimed on Vercel by another account at phase 11 setup time)
- Download CTA points at the GitHub Releases DMG asset (rationale: no commerce, no email capture; GitHub Releases is already the distribution channel via Sparkle)
- No public changelog (rationale: 2026-04-25 scope reduction — release notes stay a developer-internal artifact; phase 15 was built and then reverted)
- Build-time sidebar codegen for docs from `page.mdx` `doc` exports (rationale: drift-by-construction — sidebar can't go out of sync with actual MDX files because it's regenerated from them)

**Issues Resolved:**

- Vercel slug collision discovered at phase 11 — resolved by switching to `ps-transcribe-web.vercel.app` and updating PROJECT.md to flag the discrepancy
- LAND-06 originally specified a `/changelog` link in nav; amended on 2026-04-25 after phase 15 was reverted
- Phase 14 mock had macOS 14 references; D-10 fix during phase 14 execution corrected to macOS 26 (the actual minimum)
- MDX plugin loader fix during 14-04 execution (resolved before plan close)

**Issues Deferred:**

- Custom domain (v1.2 candidate)
- Docs search (Cmd+K modal) — deferred per scope
- Localization — English-only this milestone
- Analytics stack decision — Vercel default only

**Technical Debt Incurred:**

- Production hosted on Vercel subdomain (`ps-transcribe-web.vercel.app`) instead of intended `ps-transcribe.vercel.app` — slug fallback noted in PROJECT.md, expected to be superseded by a custom domain in v1.2
- OWNER placeholder still present in `SUFeedURL` and `release-dmg.yml` (carried forward from v1.0)
- No automated visual regression testing for the website yet — UAT was human-driven
