# Phase 6 -- UI Design Contract

> Visual and interaction contract for Phase 6: Live LLM Analysis.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

This is a native macOS SwiftUI app. There is no web component library, no shadcn, and no Tailwind. Design tokens are declared as `Color` extensions in `TranscriptView.swift` and are used across all existing views.

| Property | Value |
|----------|-------|
| Tool | none (native SwiftUI) |
| Preset | not applicable |
| Component library | SF Symbols (system icons) |
| Icon library | SF Symbols -- system-provided |
| Font | .system (San Francisco, macOS default) |

Source: codebase scan of `TranscriptView.swift`, `ControlBar.swift`, `ContentView.swift`

## Spacing Scale

Tokens declared as Swift constants where needed. Values match existing patterns extracted from the codebase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline dot indicators |
| sm | 8px | VStack spacing within bars, button inner VStack, list row spacing |
| md | 16px | Bubble/card padding, section padding, ScrollView content padding |
| lg | 16px | Bar horizontal padding, element separation |
| xl | 24px | Panel section internal spacing |
| 2xl | 32px | Major panel section gaps |
| 3xl | 48px | Empty state padding (matches existing LibrarySidebar pattern) |

Note: `md` and `lg` both resolve to 16px. `md` signals internal component padding; `lg` signals inter-element separation. They share the same value intentionally -- 16px is the established horizontal padding from `ControlBar.swift`.

Exceptions:
- Analysis panel toggle button touch target: 28x28pt minimum (matches existing mic icon frame in ControlBar)
- Analysis panel width: fixed 220pt minimum, 260pt ideal (mirrors transcript bubble max-width of 260pt)

Source: Extracted from `ControlBar.swift` (.padding(.horizontal, 16), .padding(.vertical, 8)), `TranscriptView.swift` (.padding(10), spacing: 12), `LibrarySidebar.swift` (.padding(48))

## Typography

All sizes use `.system` font (San Francisco). No custom typeface is introduced in this phase. Maximum 2 weights: regular (400) and semibold (600). Section headings are visually distinguished from labels via `.textCase(.uppercase)` and `.tracking(0.8)`, not a third weight.

| Role | Size | Weight | Line Height | Modifiers |
|------|------|--------|-------------|-----------|
| Body | 12pt | regular (400) | 1.5 (SwiftUI default) | none |
| Label | 11pt | semibold (600) | 1.2 | none |
| Caption | 10pt | regular (400) | 1.2 | none |
| Section heading | 10pt | semibold (600) | 1.2 | `.textCase(.uppercase)`, `.tracking(0.8)` |

Notes:
- Section headings in the analysis panel (Summary, Action Items, Key Topics) use 10pt/semibold/uppercase/tracking:0.8 -- matching the visual treatment of speaker labels in `UtteranceBubble` while staying within the 2-weight limit
- Action item text uses 12pt/regular body role
- "Updating..." status label uses 10pt/regular caption role
- No new font sizes are introduced -- all roles map to existing codebase patterns

Source: `TranscriptView.swift` (speaker label: size 10, weight .bold, tracking 0.8; body: size 12; timestamp: size 10), `ControlBar.swift` (button label: size 11 weight .semibold; caption: size 10). Bold (700) consolidated to semibold (600) to satisfy 2-weight maximum; visual distinction preserved via uppercase + tracking modifiers.

## Color

All tokens are defined in `TranscriptView.swift` `Color` extension. No new color tokens are introduced in this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1A1818` (`Color.bg0`) | App background, main content area |
| Secondary (30%) | `#242120` (`Color.bg2`) | Sidebar, analysis panel background |
| Card surface | `#2E2B29` (`Color.bg1`) | Analysis section cards, bubble backgrounds, button backgrounds |
| Accent (10%) | `#C4A0FF` (`Color.accent1`) | Toggle button active state, section heading left-border accent stripe, pulsing status dot during live update |
| Destructive | `#E85B5B` (`Color.recordRed`) | Not used in this phase |

Accent reserved for:
- Analysis panel toggle button: active/enabled state highlight
- Section left-border stripe (3pt width, same pattern as `UtteranceBubble` speaker accent stripe)
- Live update pulsing dot indicator (4pt circle, same as `VolatileIndicator` dot)

Foreground tokens:
- `Color.fg1` (`#F0EDE8`) -- primary content text (action item text, summary body)
- `Color.fg2` (`#8A8480`) -- secondary labels (section sub-labels, key topic chips text)
- `Color.fg3` (`#5C5854`) -- muted/tertiary (timestamps, "No analysis yet" empty state)

Source: `TranscriptView.swift` Color extension (all tokens), confirmed usage in `ControlBar.swift`, `LibrarySidebar.swift`

## Component Inventory

### AnalysisPanel (new)

A scrollable right-side panel displaying three sections: Summary, Action Items, Key Topics.

**Structure:**
```
VStack(spacing: 0) {
  // Panel header
  HStack {
    Text("Analysis")       // 10pt semibold uppercase tracking:0.8 fg2
    Spacer()
    StatusDot()            // 4pt circle, accent1, pulsing when update in-flight
  }
  .padding(.horizontal, 16)
  .padding(.vertical, 8)

  Divider()

  // Content
  ScrollView {
    VStack(spacing: 24) {
      AnalysisSectionCard(title: "Summary", content: summary)
      AnalysisSectionCard(title: "Action Items", items: actionItems)
      AnalysisSectionCard(title: "Key Topics", topics: keyTopics)
    }
    .padding(16)
  }
}
.frame(minWidth: 220, idealWidth: 260, maxWidth: 300)
.background(Color.bg2)
.overlay(Divider(), alignment: .leading)  // left border separator from transcript
```

**States:**
1. Empty (no analysis yet, Ollama connected but recording just started):
   - SF Symbol `brain` at 24pt, `Color.fg3`
   - Text "No analysis yet" at 12pt `Color.fg2`
   - Text "Analysis will appear as the conversation progresses" at 11pt `Color.fg3`
   - No section cards shown

2. Loading (first analysis in-flight):
   - `ProgressView().controlSize(.mini).tint(Color.accent1)` -- matches existing `ControlBar` pattern
   - Text "Analyzing..." at 10pt `Color.fg2`
   - Centered in panel body

3. Live (analysis data present, may be refreshing):
   - All three section cards visible
   - Pulsing `accent1` 4pt dot in header when update in-flight
   - Dot hidden when idle

4. Stale/review (past session with saved analysis):
   - Same as Live state but no pulsing dot
   - No "updating" indicator

### AnalysisSectionCard (new)

Reusable card for each section in the panel.

**Structure:**
```
VStack(alignment: .leading, spacing: 8) {
  // Section header -- 10pt semibold uppercase tracking:0.8 (no bold needed)
  Text(title)
    .font(.system(size: 10, weight: .semibold))
    .textCase(.uppercase)
    .tracking(0.8)
    .foregroundStyle(Color.accent1)

  // Left accent stripe + content area
  HStack(spacing: 0) {
    // Accent stripe -- identical to UtteranceBubble left border
    Rectangle()
      .fill(Color.accent1.opacity(0.4))
      .frame(width: 3)

    content
      .padding(.leading, 8)
  }
}
.padding(16)
.background(Color.bg1.opacity(0.7))
.clipShape(RoundedRectangle(cornerRadius: 10))
.overlay(RoundedRectangle(cornerRadius: 10).stroke(Color.white.opacity(0.06)))
```

### Action Items List

Each action item rendered as a row:
```
HStack(alignment: .top, spacing: 6) {
  Image(systemName: "square")  // or "checkmark.square" if checked
    .font(.system(size: 12))
    .foregroundStyle(Color.fg3)
  Text(item)
    .font(.system(size: 12))
    .foregroundStyle(Color.fg1)
    .fixedSize(horizontal: false, vertical: true)
}
```

Row spacing between action items: 8px (`sm` token).

### Key Topics Chips

Displayed as a flowing `LazyVGrid` or wrapping `HStack` of pill shapes:
```
Text(topic)
  .font(.system(size: 10, weight: .semibold))
  .foregroundStyle(Color.fg2)
  .padding(.horizontal, 8)
  .padding(.vertical, 4)
  .background(Color.bg0.opacity(0.8))
  .clipShape(Capsule())
  .overlay(Capsule().stroke(Color.white.opacity(0.08)))
```

### Analysis Toggle Button (in ControlBar)

Added to ControlBar when Ollama is connected. Hidden when Ollama is not connected (LLMA-06, D-05).

```
Button(action: toggleAnalysisPanel) {
  Image(systemName: showAnalysis ? "brain.fill" : "brain")
    .font(.system(size: 14))
    .foregroundStyle(showAnalysis ? Color.accent1 : Color.fg2)
    .frame(width: 28, height: 28)
}
.buttonStyle(.plain)
.help(showAnalysis ? "Hide Analysis" : "Show Analysis")
```

Placement: Trailing edge of ControlBar HStack, before/near the session buttons. Only rendered when `ollamaState.connectionStatus == .connected`.

## Layout Contract

### Three-column layout during recording (D-01)

When analysis panel is visible and a session is active or a past session with saved analysis is selected:

```
NavigationSplitView (sidebar) {
  LibrarySidebar   // 180-300pt wide (existing)
} detail: {
  HStack(spacing: 0) {
    TranscriptView()        // .frame(maxWidth: .infinity)
    AnalysisPanel()         // minWidth: 220, idealWidth: 260, maxWidth: 300
  }
}
```

When panel is hidden or no analysis is available:
- Detail area is `TranscriptView` alone (current behavior, unchanged)

Transition: `.animation(.spring(duration: 0.35, bounce: 0.15))` matching existing ControlBar session button animation.

### Panel width behavior
- Panel does NOT resize with a drag handle in this phase (deferred)
- Fixed to idealWidth: 260pt
- Panel appears/disappears via `showAnalysisPanel` toggle with spring animation

## Interaction Contract

### Toggle button behavior

| State | Ollama connected | Recording active | Past session with analysis | Toggle visible |
|-------|-----------------|------------------|---------------------------|----------------|
| Hidden | No | Any | Any | No |
| Shown, enabled | Yes | Yes | - | Yes |
| Shown, enabled | Yes | No | Yes (has saved analysis) | Yes |
| Shown, disabled | Yes | No | No (no analysis data) | Yes, opacity 0.4 |

### Analysis update cycle

1. `handleNewUtterance()` in ContentView increments utterance counter
2. When counter reaches threshold (every 8 utterances) AND 30-second cooldown has elapsed AND no request is in-flight:
   - Set `isUpdating = true` in AnalysisState (shows pulsing dot)
   - Call `OllamaService.generate(prompt:model:)` with full transcript
3. On response: parse into summary, actionItems, keyTopics -- update AnalysisState
4. Set `isUpdating = false`, reset utterance counter, record timestamp
5. On error (timeout, Ollama unavailable): silently skip -- do not show error in panel, do not interrupt recording

### Keyboard shortcut

None for this phase. Toggle is mouse/trackpad only via ControlBar button.

## Persistence Contract

When session ends, `TranscriptLogger` appends the following markdown to the transcript file (D-12, D-13):

```markdown
## Analysis

### Summary

{summary text}

### Action Items

- [ ] {item 1}
- [ ] {item 2}

### Key Topics

{topic1}, {topic2}, {topic3}
```

This is written once at session end using the final `AnalysisState` snapshot. If no analysis was generated during the session (Ollama unavailable or session too short), the `## Analysis` section is omitted entirely.
