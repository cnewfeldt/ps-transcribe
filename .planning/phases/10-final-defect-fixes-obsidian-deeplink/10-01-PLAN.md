**Fix 1 -- Speaker label mapping (D-04):**

At ContentView.swift line 449, replace the ternary:
```swift
let speakerLabel = removed.speaker == .you ? "You" : "Them"
```

With a switch that handles all three Speaker cases:
```swift
let speakerLabel: String
switch removed.speaker {
case .you:             speakerLabel = "You"
case .them:            speakerLabel = "Them"
case .named(let lbl):  speakerLabel = lbl
}
```

This maps `.named("Speaker 2")` to the string `"Speaker 2"`, which matches the `**Speaker 2** (HH:mm:ss)` header format in the markdown file. The old ternary collapsed all non-.you speakers to "Them", so removing a named speaker's utterance could never match the header line.

**Fix 2 -- Crash recovery session type (D-05/D-06):**

At ContentView.swift around line 227, inside the crash recovery `.task` block where `LibraryEntry` is constructed, replace:
```swift
sessionType: .callCapture,
```

With session type inference based on vault paths. Before the `LibraryEntry(...)` constructor (but inside the `if !existsInLibrary` block), add:
```swift
let recoveredType: SessionType
if checkpoint.transcriptPath.hasPrefix(settings.vaultVoicePath) {
    recoveredType = .voiceMemo
} else {
    recoveredType = .callCapture
}
```

Then use `recoveredType` in the constructor:
```swift
sessionType: recoveredType,
```

Note: `settings` is already accessible in ContentView (it's used at line 53-54 for `settings.notionDatabaseID`). The `.task` block runs on `@MainActor` since ContentView is a `View`, so reading `settings.vaultVoicePath` and `settings.vaultMeetingsPath` is safe. Per D-06, default is `.callCapture` when the path matches neither vault -- which the else branch handles.
