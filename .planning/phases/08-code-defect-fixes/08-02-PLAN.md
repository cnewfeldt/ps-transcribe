<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem -> LibraryEntryRow | File-exists check reads disk state; result cached to prevent per-render I/O |
| YAML frontmatter -> external tools | Frontmatter tags consumed by Obsidian/external tools; correct tag identity matters |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | I (Information Disclosure) | print() on error paths | mitigate | Replace print() with os.Logger using `privacy: .public` only for error.localizedDescription (no transcript content, no file paths beyond what localizedDescription contains). os.Logger respects system log levels and does not write to world-readable files. |
| T-08-04 | T (Tampering) | source/pstranscribe tag | accept | Frontmatter tag is informational metadata written to user-owned markdown files. Incorrect tag ("tome") is a branding defect, not a security issue. Fix is mechanical single-line change. |
| T-08-05 | D (Denial of Service) | fileExists cached in @State | accept | Stale cache (file deleted after onAppear) shows no badge until row re-appears. Acceptable UX tradeoff vs per-render filesystem I/O. |
</threat_model>
