# Phase 8 -- UI Design Contract

> Visual and interaction contract for Phase 8: Code Defect Fixes.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

> (2) an "Incomplete" yellow badge in LibraryEntryRow for crash-recovered sessions.

### New Tokens (Phase 8 -- Speaker Badge Colors)

Add to the same `extension Color` block in `TranscriptView.swift`:

| Token | Value | Usage |
|-------|-------|-------|
| `speakerTeal` | `Color(red: 0.60, green: 0.85, blue: 0.75)` | `.named` Speaker 2 accent bar and label |
| `speakerAmber` | `Color(red: 0.95, green: 0.75, blue: 0.45)` | `.named` Speaker 3 accent bar and label |

Additional named speakers beyond index 3 cycle: `[speakerTeal, speakerAmber][(n - 2) % 2]`.
Fallback for unrecognized label format: `.fg2` (same as "Them").

### 60/30/10 Distribution

| Role | Tokens | Approximate % |
|------|--------|---------------|
| Dominant surface | `bg0`, `bg2` | 60% |
| Secondary (cards, sidebar) | `bg1` | 30% |
| Accent | `accent1`, `speakerTeal`, `speakerAmber` | 10% -- see reserved list below |

Accent reserved for:
- "You" speaker label text and left accent bar in `UtteranceBubble`
- Selected left stripe in `LibraryEntryRow`
- File path link text in `LibraryEntryRow`
- "Speaker 2" label text and left accent bar (`speakerTeal`)
- "Speaker 3" label text and left accent bar (`speakerAmber`)

Destructive: `recordRed` -- reserved for missing-file badge, error mic state, delete action. Do not use for informational states.

## Component Inventory

### Modified Components (Phase 8)

#### 1. Speaker enum (Models.swift)

No visual output directly. Data model change that unlocks visual changes below.

- Add `.named(String)` case with manual Codable conformance
- Backward-compat decoder handles legacy `"you"` / `"them"` raw strings

#### 2. UtteranceBubble (TranscriptView.swift)

Current: `accentColor` is binary -- `.accent1` for `.you`, `.fg2` for `.them`.
Target: Three-branch switch.

```swift
private var accentColor: Color {
    switch utterance.speaker {
    case .you:              return .accent1
    case .them:             return .fg2
    case .named(let label): return Self.namedSpeakerColor(for: label)
    }
}

private static func namedSpeakerColor(for label: String) -> Color {
    let palette: [Color] = [.speakerTeal, .speakerAmber]
    if let n = label.split(separator: " ").last.flatMap({ Int($0) }), n >= 2 {
        return palette[(n - 2) % palette.count]
    }
    return .fg2
}
```

Speaker label text (line 69): Replace hardcoded `"You"` / `"Them"` with:
```swift
private var speakerDisplayName: String {
    switch utterance.speaker {
    case .you:              return "You"
    case .them:             return "Them"
    case .named(let label): return label
    }
}
```

Style: same 10pt bold uppercase 0.8 tracking as existing speaker labels. No new font style.

#### 3. VolatileIndicator (TranscriptView.swift)

Current: Only handles `.you` and `.them`.
Target: Add `.named` to `accentColor` switch using same `namedSpeakerColor` helper.
No layout changes needed.

#### 4. LibraryEntryRow (LibraryEntryRow.swift)

Two changes:

**A. Incomplete badge (D-03)**

Add BEFORE the existing missing-file badge check. Show only when `!entry.isFinalized`. Do NOT show missing-file badge when `!entry.isFinalized` (incomplete overrides missing).

```
Icon: clock.badge.exclamationmark.fill (SF Symbols)
Color: Color.yellow.opacity(0.85)
Font: .system(size: 10)
Padding: .padding(.top, 4)
Tooltip: "Session was interrupted -- transcript may be incomplete"
```

Badge display rule:
- `!entry.isFinalized` --> show Incomplete badge (yellow), suppress missing-file badge
- `entry.isFinalized && !fileExists` --> show missing-file badge (red)
- `entry.isFinalized && fileExists` --> no badge

**B. File-exists caching (D-06)**

Replace inline `FileManager.default.fileExists(atPath:)` at line 80 with `@State private var fileExists: Bool = true`, populated in `.onAppear`. No visual change -- purely a performance fix.

#### 5. typeIconName helper (LibraryEntryRow.swift line 144)

Current: `exclamationmark.circle` for `!isFinalized`. Keep as-is -- the icon already correctly distinguishes incomplete entries. No change needed.

## Interaction Contract

### Named Speaker Bubbles

- Same bubble layout as `.you` and `.them` -- no layout change
- `.named` speakers align LEFT (same as `.them`) -- they are third-party voices, not the local user
- Left accent bar uses `namedSpeakerColor(for:)` -- 3pt width, same as existing
- Speaker label uppercase 10pt bold -- same style as `.you` / `.them`

### Incomplete Badge (LibraryEntryRow)

- Informational only -- tapping does nothing
- Row remains fully interactive: rename, send to Notion, load transcript all enabled
- Badge appears in the trailing icon slot (right side of row, alongside where missing-file badge appears)
- Tooltip via `.help(...)` -- macOS native tooltip on hover

### File-Exists Check Timing

- Check runs once on `.onAppear`
- Refreshes when row scrolls out of view and back (natural SwiftUI `.onAppear` behavior)
- No manual refresh button or forced re-check needed
