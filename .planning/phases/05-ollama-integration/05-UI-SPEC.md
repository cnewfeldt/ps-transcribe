## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | icon-to-text gaps within rows |
| sm | 8px | spacing between sibling controls in a row |
| md | 12px | horizontal/vertical padding for control groups (matches ControlBar) |
| lg | 16px | horizontal padding for SettingsView rows (matches existing Section rows) |
| xl | 20px | vertical gap between icon and heading in sheet (matches OnboardingView) |
| 2xl | 28px | sheet outer padding (matches OnboardingView .padding(28)) |

Exceptions:
- SettingsView Section rows use `.font(.system(size: 12))` spacing -- vertical spacing is handled by `.formStyle(.grouped)` system behavior, not explicit padding
- Status dot size is 9pt (SF Symbol `.system(size: 9)`) per D-07 pattern in RESEARCH.md -- not a spacing token

All fonts use `.system()` (San Francisco). No custom typeface is introduced in this phase.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 12pt | regular (400) | system default (~1.4) | Settings row labels, picker options, Browse Models button |
| Caption | 11pt | regular (400) | system default (~1.4) | Section sublabels (e.g. vault path display, "run ollama pull" hint) |
| Status text | 12pt | regular (400) | system default (~1.4) | Connection status label next to colored dot |
| Sheet heading | 16pt | semibold (600) | 1.2 | OllamaModelBrowseSheet title, matches OnboardingView heading pattern |

Source: measured from SettingsView.swift (12pt medium labels, 11pt secondary sublabels) and OnboardingView.swift (16pt semibold headings).

## Color

All colors reference the established token set from `TranscriptView.swift`.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `Color.bg0` / `Color.bg1` | #1A1818 / #2E2B29 | Settings form background, sheet background |
| Secondary (30%) | `Color.bg1.opacity(0.7)` | #2E2B29 at 70% | Grouped form section cells, sheet card backgrounds |
| Accent (10%) | `Color.accent1` | #C4A0FF | Reserved elements listed below |
| Connection: connected | `Color.green` (system) | system green | Connected status dot only |
| Connection: not running | `Color.recordRed` | #E85B5B | Not running status dot only |
| Connection: not found | `Color.fg3` | #5C5854 | Not found status dot (matches disabled/tertiary intent) |
| Destructive | `Color.recordRed` | #E85B5B | Destructive actions only -- none in this phase |

**Accent (`Color.accent1 #C4A0FF`) reserved for:**
- ProgressView tint in OllamaModelBrowseSheet loading state
- Selected model highlight in Picker (via `.tint`)
- Active row indicator (3pt left edge on selected model in browse sheet, matches utterance bubble pattern)

**Foreground hierarchy (existing tokens, used as-is):**
- Primary text: `Color.fg1` (#F0EDE8)
- Secondary text: `Color.fg2` (#8A8480)
- Tertiary / hint text: `Color.fg3` (#5C5854)

Source: D-07 (CONTEXT.md) specifies green/red/gray dot semantics. Token values from TranscriptView.swift Color extension.

## Component Inventory

New components introduced in this phase:

### 1. Ollama Settings Section (inline in SettingsView)

Location: `SettingsView.swift` -- `Section("Ollama")` inserted after `Section("Audio Input")`

```
Section("Ollama") {
  [status row]          HStack: 9pt dot + 12pt status label + Spacer
  [model picker]        Picker("Model", ...) -- visible only when .connected && !models.isEmpty
  [empty hint]          11pt caption -- visible only when .connected && models.isEmpty
  [browse button]       Button("Browse Models") -- visible only when .connected
}
```

States:
- `.notFound`: status row only (gray dot + "Not found"), no picker, no browse button
- `.notRunning`: status row only (red dot + "Not running"), no picker, no browse button
- `.connected, models empty`: status row (green dot + "Connected") + 11pt hint "No models found -- run `ollama pull llama3.2:3b` in Terminal"
- `.connected, models present`: status row + Picker + Browse Models button

### 2. OllamaModelBrowseSheet

Location: `PSTranscribe/Sources/PSTranscribe/Views/OllamaModelBrowseSheet.swift` (new file)

Layout: Sheet presented modally, 400x300pt minimum size.

```
VStack(spacing: 0) {
  [heading]     16pt semibold "Downloaded Models"
  [subhead]     13pt secondary "Select a model to use for analysis"
  [list]        List of OllamaModel rows, or ProgressView while loading, or error inline
  [close]       Button "Done" -- bottom trailing
}
.padding(24)
```

Model row (when loaded):
```
HStack {
  VStack(alignment: .leading, spacing: 2) {
    Text(model.name)    12pt fg1
    Text(model.details.parameterSize + " · " + model.details.quantizationLevel)
                        11pt fg2
  }
  Spacer
  [checkmark]   Image(systemName: "checkmark") -- accent1, visible when model.name == selectedModel
}
```

Loading state: `ProgressView()` centered, tinted accent1, caption "Loading models..."

Error state (fetch failed): inline 11pt fg2 "Unable to connect to Ollama. Make sure Ollama is running."

## Interaction Contracts

### Status check trigger (D-02)
- `OllamaState.refresh()` called in `.onAppear` of SettingsView
- No polling -- single check per appearance
- `isCheckingConnection` drives a `ProgressView().controlSize(.mini)` next to the status dot during the 2-second window

### Model picker behavior
- Picker only rendered when `connectionStatus == .connected && !models.isEmpty`
- Selection bound to `AppSettings.selectedOllamaModel`
- If stored value is empty or not in current model list: auto-select `llama3.2:3b` if present, else first model
- No "None" option -- a model is always selected when Ollama is connected

### Browse sheet behavior
- Sheet triggered by "Browse Models" button (only visible when .connected)
- Sheet calls `fetchModels()` in `.task {}` on appear -- auto-cancelled on dismiss
- While loading: ProgressView centered
- On error: inline error message (no sheet dismissal)
- Tapping a model row updates `AppSettings.selectedOllamaModel` immediately (no confirm step)
- "Done" button dismisses sheet

### Disabled state (no Ollama)
- Picker and Browse Models button are absent (not disabled + grayed) when not connected
- Status-only row provides clear reason without action buttons (D-03)

## Animation + Transition Contracts

- Status dot color change: no explicit animation -- SwiftUI default state update
- Picker appearance (connected -> disconnected): `.animation(.spring(duration: 0.25), value: connectionStatus)` on the containing VStack
- Sheet presentation: default SwiftUI sheet animation
- Model list appearance: no animation -- list loads from async fetch

---

## Accessibility

- Status dot must not rely on color alone -- dot + text label always present (D-07)
- Picker uses standard SwiftUI Picker -- VoiceOver reads label + selection
- Browse Models button has no `.help()` tooltip required (action is clear from label)
- Model rows in sheet are tappable full-width (no icon-only targets)
- SF Symbols used at ≥9pt -- never smaller
