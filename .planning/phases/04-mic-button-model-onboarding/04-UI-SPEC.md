Semantic colors specific to this phase:
- **Recording green:** `Color.green` (SwiftUI system green) -- mic icon tint + pulsing ring stroke during recording state only. Source: CONTEXT.md D-02, RESEARCH.md SF Symbol selection.

Foreground tokens:
- `Color.fg1` (#F0EDE8) -- primary text, button labels
- `Color.fg2` (#8A8480) -- secondary text, mic idle state icon

## Component Inventory

### MicButton (new -- ControlBar.swift)

A private `MicButton` struct added below `PulsingDot` in ControlBar.swift. Follows the PulsingDot file-scoped component pattern.

**States:**

| State | Trigger | Icon | Icon Color | Ring |
|-------|---------|------|------------|------|
| idle | `isRecording == false`, no errors | `mic.fill` | `Color.fg2` | none |
| recording | `isRecording == true` | `mic.fill` | `Color.green` | green pulsing, radiates outward |
| error | `hasError == true` (any of 4 conditions) | `mic.slash` | `Color.recordRed` | none |

Error overrides recording: if `hasError == true`, error state renders regardless of `isRecording`.

**Dimensions:**
- Icon font size: 24pt (`.font(.system(size: 24))`)
- Icon frame: 44 x 44pt
- Ring outer frame: 52 x 52pt
- Ring stroke width: 2.5pt
- Ring color: `Color.green.opacity(0.7)`

**Tap behavior by state:**
- idle: calls `onStartLastUsed()` -- starts the last-used session type (default: Call Capture)
- recording: calls `onStop()` -- second stop mechanism alongside existing stop bar
- error: calls `openSettings()` -- `NSApp.sendAction(Selector("showSettingsWindow:"))` pattern

**Tooltip (error state only):**
- `.help(activeErrorsJoined)` where `activeErrorsJoined` is newline-separated string of all active errors
- If newline rendering fails in QA, fall back to " | " separator
- Source: CONTEXT.md D-09, RESEARCH.md pitfall 3

**Animation (recording state only):**
- Type: radar ping -- scale outward with opacity fade, non-reversing repeat
- Scale: 1.0 to 1.8
- Opacity: 0.6 to 0.0
- Duration: 1.4s
- Easing: `.easeOut`
- Repeat: `.repeatForever(autoreverses: false)`
- Source: RESEARCH.md Pattern 1 + code example

**Button style:** `.buttonStyle(.plain)` + `.focusable(false)` -- matches all existing ControlBar buttons.

### ControlBar layout restructure

**Idle state (new layout):**
```
[Call Capture btn (flex)] [MicButton 44pt] [Voice Memo btn (flex)]
```
Gear button moves to fixed position -- top-right corner of ControlBar, persistent across recording states.

**Recording state (new layout):**
```
[Stop Recording bar (full-width, existing)]
[MicButton centered -- recording ring active]
```
MicButton is always rendered in the view hierarchy (never conditionally inserted/removed) to preserve `@State` animation continuity. Source: RESEARCH.md pitfall 1.

**ControlBar height:** Both idle and recording branches must produce identical total height. Use a fixed outer frame on the ControlBar VStack. Recording state adds MicButton row -- set a stable frame height that accommodates both. Target: ~88pt total (existing ~44pt + ~44pt MicButton row). Source: CONTEXT.md D-06 ("control bar height grows to accommodate"), RESEARCH.md pitfall 4.

### OnboardingView additions

**Failure state (new branch in download step):**

| Element | Value |
|---------|-------|
| Icon | `xmark.circle` in `Color.recordRed`, 40pt light weight |
| Title | "Download Failed" -- 16px semibold |
| Body | Error detail from `modelStatus` string -- 13px regular, `.secondary` |
| Button | "Try Again" -- 13px medium weight, `Color.accent1` background, calls `onRetry()` |

Failure detection: `downloadFailed` computed property -- `!modelsReady && modelStatus.lowercased().contains("failed")`.

**Success state (existing -- augmented):**
- Icon: `checkmark.circle` in `Color.green` (already exists)
- Title: "Speech Model Ready" (already exists)
- Body text: explicit confirmation copy -- "The on-device speech model is installed. You're ready to start transcribing." (already exists)
- Button: "Get Started" enabled -- `Color.accent1` background (already exists, just needs to be clearly enabled)

**No-dismiss constraint:** The "Get Started" button remains `.disabled(!modelsReady)`. There is no skip/dismiss path on the model download step. Source: CONTEXT.md D-13, RESEARCH.md anti-pattern warning.

**New `onRetry` callback parameter on OnboardingView:**
```swift
let onRetry: () -> Void
```
Wired in ContentView to `Task { await transcriptionEngine?.prepareModels() }`.

## Interaction States Summary

### MicButton full state matrix

| Condition | State | Visual |
|-----------|-------|--------|
| `modelsReady == false`, no errors | disabled idle | `mic.fill`, `Color.fg2`, opacity 0.4 |
| `modelsReady == true`, not recording, no errors | idle | `mic.fill`, `Color.fg2`, opacity 1.0 |
| `isRecording == true`, no errors | recording | `mic.fill`, `Color.green`, green pulsing ring |
| `hasError == true` (any condition) | error | `mic.slash`, `Color.recordRed`, `.help(errors)` |

Note: error overrides recording. If recording fails and `hasError` becomes true mid-session, the mic icon switches to error state immediately.

### OnboardingView state matrix

| Condition | Step | Visual |
|-----------|------|--------|
| `currentStep < 2` | Info slide 1 or 2 | SF Symbol icon + title + body + "Next" button |
| `currentStep == 2`, `!modelsReady`, `!downloadFailed` | Downloading | `arrow.down.circle` (`accent1`) + progress bar + status text |
| `currentStep == 2`, `downloadFailed` | Failed | `xmark.circle` (`recordRed`) + error text + "Try Again" button |
| `currentStep == 2`, `modelsReady` | Success | `checkmark.circle` (`green`) + success text + "Get Started" button (enabled) |

## Implementation Notes for Executor

1. **MicButton is always rendered** -- never wrapped in `if state == .recording`. Pass `state` as a parameter and branch internally to preserve `@State` animation continuity.

2. **`prepareModels()` catch block** must reset `self.asrManager = nil` and `self.vadManager = nil` before setting `lastError`. This unblocks the retry guard. Source: RESEARCH.md open question 2 (resolved).

3. **`activeErrors: [String]`** computed property on `TranscriptionEngine` aggregates the four D-08 conditions. Join with `"\n"` for `.help()`. Do not replace `lastError: String?` -- it is used throughout the codebase. Source: RESEARCH.md Pattern 2.

4. **ControlBar `onStartLastUsed` callback** -- new parameter. Calls `onStartCallCapture` or `onStartVoiceMemo` based on `AppSettings.lastUsedSessionType`. `AppSettings` gets a new `lastUsedSessionType: SessionType` property persisted via `didSet`/UserDefaults. Source: CONTEXT.md D-03, RESEARCH.md code example.

5. **Settings window from error tap** -- reuse the exact `NSApp.sendAction(Selector("showSettingsWindow:"))` pattern from ControlBar gear button. No custom window management. Source: RESEARCH.md "Don't Hand-Roll" table.

6. **WaveformView.swift deleted** -- ContentView line 269 `WaveformView(isRecording:audioLevel:)` removed. `audioLevel` parameter on ControlBar remains available for optional ring intensity modulation. Source: CONTEXT.md D-15, D-16.
