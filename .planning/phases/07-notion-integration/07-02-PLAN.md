# Plan 07-02: Settings Notion Section

## Objective

Add a "Notion" section to SettingsView where the user configures their API key (stored in Keychain via KeychainHelper) and target database ID. The section shows connection status, workspace name, and database title after validation. When fully configured, it enables the "Send to Notion" context menu item in library rows (Plan 07-03).

## Reference

- Design: `docs/plans/2026-04-05-notion-integration-design.md` (Settings UI States section)
- Existing Settings pattern: `PSTranscribe/Sources/PSTranscribe/Views/SettingsView.swift`

## Tasks

### Task 1: AppSettings + SettingsView Notion section

**AppSettings changes (`AppSettings.swift`):**

Add these properties:
```swift
var notionDatabaseID: String {
    didSet { UserDefaults.standard.set(notionDatabaseID, forKey: "notionDatabaseID") }
}
```

In `init()`:
```swift
self.notionDatabaseID = defaults.string(forKey: "notionDatabaseID") ?? ""
```

Note: API key is NOT in AppSettings -- it lives in Keychain, accessed only through NotionService.

**SettingsView changes (`SettingsView.swift`):**

Add a `Section("Notion")` after the existing "Audio Input" section. The section needs local `@State` for the validation flow since it's async:

```swift
@State private var notionAPIKeyInput: String = ""
@State private var notionWorkspaceName: String? = nil
@State private var notionDatabaseName: String? = nil
@State private var notionStatus: NotionConfigStatus = .notConfigured
@State private var notionError: String? = nil
@State private var isValidatingNotion: Bool = false
```

Where `NotionConfigStatus` is a private enum: `.notConfigured`, `.connected`, `.fullyConfigured`.

**Four UI states:**

1. **Not configured** (no API key in Keychain):
   - SecureField for API key with "Paste your integration token" placeholder
   - "Save" button that calls `notionService.setApiKey()` then `testConnection()`
   - Inline help text: "Create an integration at notion.so/my-integrations"

2. **Validating** (after Save or on appear if key exists):
   - ProgressView spinner next to "Checking connection..."

3. **Connected, no database** (API key valid, database ID empty):
   - Green dot + workspace name
   - TextField for database ID/URL with "Paste database URL or ID"
   - Validate button that calls `notionService.validateDatabase()`
   - "Remove" button next to API key that calls `deleteApiKey()` and resets to state 1

4. **Fully configured** (both valid):
   - Green dot + "Connected to {workspaceName}"
   - Database: "{databaseName}" with "Change" button
   - "Remove" button for API key

**Database ID/URL parsing:**
Users will paste a full Notion URL like `https://www.notion.so/myworkspace/abc123def456?v=...`. The database ID is the 32-char hex string in the path. Parse it:
```swift
// Extract 32-char hex ID from Notion URL or use raw input if already an ID
private func parseDatabaseID(_ input: String) -> String {
    // Match 32 hex chars (with or without dashes) in the URL path
    let pattern = #"([a-f0-9]{32}|[a-f0-9]{8}-[a-f0-9]{4}-[a-f0-9]{4}-[a-f0-9]{4}-[a-f0-9]{12})"#
    // ...
}
```

**NotionService instance:**
SettingsView needs a `NotionService` instance. Since NotionService is an actor, the simplest approach is to pass it from PSTranscribeApp → SettingsView. Add a `notionService: NotionService` parameter to SettingsView and instantiate it once in PSTranscribeApp (alongside settings).

Update `PSTranscribeApp.swift`:
```swift
@State private var notionService = NotionService()
// ...
Settings {
    SettingsView(settings: settings, updater: updaterController.updater, notionService: notionService)
}
```

**On appear:**
If an API key exists in Keychain, auto-validate on open (`testConnection()` then `validateDatabase()` if database ID is set). Show the validated state immediately instead of making the user click anything.

**Verify:** `swift build` clean. Manual test: open Settings, see the Notion section in its "not configured" state. Paste a valid API key, see it validate. Paste a database URL, see it resolve to a database title.
