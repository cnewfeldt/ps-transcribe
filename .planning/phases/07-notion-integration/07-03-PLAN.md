# Plan 07-03: Tag Sheet + Send Flow + Context Menu Wiring

## Objective

Wire the "Send to Notion" flow end-to-end: add `notionPageURL` to LibraryEntry for duplicate prevention, create the NotionTagSheet (tag input + send button), add context menu items to library rows, and connect everything through ContentView. This plan is `autonomous: false` because the UI needs human verification.

## Reference

- Design: `docs/plans/2026-04-05-notion-integration-design.md`
- Existing context menu: `LibraryEntryRow.swift` (.contextMenu with Rename, Show in Finder, Delete)
- Existing sheet pattern: no existing sheets in the app, but standard SwiftUI `.sheet(isPresented:)` or `.sheet(item:)`

## Tasks

### Task 1: LibraryEntry model + NotionTagSheet view

**LibraryEntry model change (`Models.swift`):**

Add `notionPageURL: String?` to `LibraryEntry`:
```swift
struct LibraryEntry: Identifiable, Codable, Sendable {
    let id: UUID
    var name: String?
    let sessionType: SessionType
    let startDate: Date
    var duration: TimeInterval
    var filePath: String
```

This is a Codable struct stored in `library.json`. Adding an optional `String?` property is backward-compatible -- existing entries decode with `nil` (Codable handles missing keys for optionals).

**Tag chips** use a simple design consistent with the app's visual language:
- `Color.accent1.opacity(0.15)` background, `Color.accent1` text, rounded rect
- "X" button on selected tags
- Tapping a previous tag moves it to selected (and removes from previous to avoid duplicates)

**Tag persistence:**
- `@AppStorage("notionPreviousTags")` stores a JSON-encoded `[String]` of the last 10 unique tags
- After a successful send, merge the newly used tags into the head of the list, deduplicate, cap at 10

**Send button behavior:**
- Disabled while `tagInput` is focused with non-empty text (hint: "press Add or Enter first")
- On tap: calls `onSend(selectedTags)`, which the parent (ContentView) handles
- The parent shows a spinner overlay on the sheet while the API call is in-flight
- On success: dismiss sheet, show brief confirmation
- On error: show error inline in the sheet, keep it open for retry

```swift

```

Make `LibraryEntry` conform to `Identifiable` (already does) for `.sheet(item:)`.

**Send function:**
```swift
private func sendToNotion(entry: LibraryEntry, tags: [String]) {
    isNotionSending = true
    notionSendError = nil
    Task {
        do {
            let markdown = try String(contentsOfFile: entry.filePath, encoding: .utf8)
            let speakers = notionService.extractSpeakers(markdown)  // sync helper
            let pageURL = try await notionService.sendTranscript(
                databaseID: settings.notionDatabaseID,
                title: entry.displayName,
                date: entry.startDate,
                duration: entry.duration,
                sourceApp: entry.sourceApp,
                sessionType: entry.sessionType == .callCapture ? "Call Capture" : "Voice Memo",
                speakers: speakers,
                tags: tags,
                transcriptMarkdown: markdown
            )
            // Store the page URL on the library entry
            await libraryStore.updateEntry(id: entry.id) { @Sendable e in
                e.notionPageURL = pageURL.absoluteString
            }
            refreshLibrary()
            notionSendEntry = nil  // dismiss sheet
            isNotionSending = false
        } catch {
            notionSendError = error.localizedDescription
            isNotionSending = false
        }
    }
}
```
