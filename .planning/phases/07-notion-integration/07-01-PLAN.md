**Notion API contract for `POST /v1/pages`:**

```json
{
  "parent": { "database_id": "..." },
  "properties": {
    "Title": { "title": [{ "text": { "content": "..." } }] },
    "Date": { "date": { "start": "2026-04-04" } },
    "Duration": { "rich_text": [{ "text": { "content": "12m 30s" } }] },
    "Source App": { "select": { "name": "Teams" } },
    "Session Type": { "select": { "name": "Call Capture" } },
    "Speakers": { "multi_select": [{ "name": "You" }, { "name": "Speaker 2" }] },
    "Tags": { "multi_select": [{ "name": "engineering" }] }
  },
  "children": [
    { "object": "block", "type": "paragraph", "paragraph": { "rich_text": [...] } }
  ]
}
```

**Headers for all requests:**
```
Authorization: Bearer {api_key}
Notion-Version: 2022-06-28
Content-Type: application/json
```

**Block conversion rules:**
- Lines matching `**Speaker** (HH:mm:ss)` → paragraph block with bold rich_text for speaker+timestamp
- Non-empty text lines → paragraph block with plain rich_text
- Empty lines → skip (Notion handles spacing)
- `---` outside frontmatter → divider block
- YAML frontmatter (between first two `---`) → strip entirely
- If total blocks > 100: first request sends blocks[0..<100], then `PATCH /v1/blocks/{page_id}/children` sends remaining in chunks of 100

**Rate limit handling:**
- On 429 response: read `Retry-After` header, wait that many seconds (default 1s), retry once. On second 429, throw.
