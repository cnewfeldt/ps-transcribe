    # Configuring your vault

    <Lede>PS Transcribe writes every transcript as a markdown file inside an Obsidian vault you pick. Four settings decide where files land and what gets filled in automatically.</Lede>

    <hr />

    ## Set the vault path

    Open **Preferences -- Vault**. The first field is the path to your Obsidian vault folder -- the same folder you'd open in Obsidian itself. Any disk works; the vault does not need to be under `~/Documents`. PS Transcribe writes files directly; Obsidian picks them up the next time the vault index refreshes.

    If you don't use Obsidian, pick any folder. The files are plain markdown -- every editor can read them.

    ## Understand the path tokens

    The meetings and memos paths accept template tokens that resolve at save time:

    ```text
    {{year}}    2026
    {{month}}   04
    {{date}}    2026-04-23
    {{title}}   the name you gave the session
    ```

    Example layout:

    ```yaml
    Meetings path:  Meetings/{{year}}
    Memos path:     Memos/{{year}}-{{month}}
    Filename:       {{date}} -- {{title}}.md
    ```

    A meeting recorded on April 23, 2026 with the title "Design review" lands at `<vault>/Meetings/2026/2026-04-23 -- Design review.md`. Rename in Obsidian afterward if you like; PS Transcribe will not touch files it has already saved.

    <Note label="On-device">
      Vault location does not affect processing. Transcription, speaker diarization, and frontmatter generation all run locally regardless of whether your vault lives on the Mac's internal drive or an external volume.
    </Note>

    ## What gets written to frontmatter

    Every saved session includes a YAML frontmatter block at the top of the file:

    ```yaml
    ---
    date: 2026-04-23
    duration: 00:32:14
    participants: [You, Sam, Priya]
    tags: [meeting, design-review]
    ---
    ```

    `date` and `duration` are filled in automatically. `participants` starts with whoever the diarizer identified and auto-labeled; rename any speaker with <Kbd>⌘</Kbd> <Kbd>E</Kbd> in-app before saving and the frontmatter updates. `tags` default to the session type (`meeting` or `memo`); add more in Preferences or per-session.

    These are the only fields PS Transcribe writes. If you want more properties, add them yourself in Obsidian -- PS Transcribe reads frontmatter on re-open but will not overwrite fields it does not own.

    ## Meetings and memos take different paths

    PS Transcribe treats meetings (dual-stream, diarized) and memos (mic-only, no diarization) as two different things and writes them to two different paths. A memo recorded during a standup does not land in your meetings archive, and a meeting does not pollute your scratch memo folder.

    You can point both paths at the same folder if you prefer a single stream; the filename tokens are enough to keep them distinct.

    # Notion property mapping

    <Lede>PS Transcribe can post each saved session as a new page in a Notion database. The transcript becomes the page body; frontmatter fields become properties -- if a matching property exists.</Lede>

    <hr />

    ## Connect the integration

    Create an internal integration in Notion at [https://www.notion.so/my-integrations](https://www.notion.so/my-integrations). The default capabilities (read / update / insert content) are sufficient. Copy the integration token.

    In PS Transcribe, open **Preferences -- Integrations -- Notion** and paste the token. The connection is verified on save; the token is stored in the macOS keychain, not in plain-text preferences.

    <Note label="No token, no traffic">
      Leave the field empty and PS Transcribe never contacts Notion. The integration is off by default and stays dormant until you connect it.
    </Note>

    ## Pick a destination database

    In Notion, open the database you want transcripts to land in. Click the three-dot menu, choose **Connections -- Add connections**, and select the integration you created. Notion restricts database access per-integration; without this step, PS Transcribe will see a "database not found" error when it tries to post.

    Back in PS Transcribe Preferences, pick the database from the dropdown. The dropdown is populated by querying the integration for pages it has access to. If your database is missing from the list, the share step has not propagated yet -- refresh in 30 seconds.

    ## Map frontmatter to Notion properties

    On save, each frontmatter field is looked up in the destination database. If a property with the same name exists and the type matches, PS Transcribe writes the value. Default mappings:

    | Frontmatter field | Notion property type | Written as |
    | --- | --- | --- |
    | `date` | Date | ISO date at the start of the recording |
    | `duration` | Number | Minutes, rounded to the nearest integer |
    | `participants` | Multi-select | One option per diarized speaker |
    | `tags` | Multi-select | `meeting` or `memo`, plus any you added |

    The session title becomes the page title. The full transcript is written to the page body as a set of toggle blocks, one per speaker turn, so the page stays scrollable even for long sessions.

    ## Fields Notion ignores

    If a frontmatter field does not have a matching property, PS Transcribe skips it silently. If a property exists but the type does not match (for example, a `Text` property where we expected `Multi-select`), the field is also skipped. Check the Notion activity feed after a save to confirm which fields landed.

    Add or rename database properties at any time; the next save picks up the changes automatically. PS Transcribe does not modify your database schema.

    # FAQ

    <Lede>Answers to the questions that come up most often. If yours is not here, file an issue on the repo.</Lede>

    <hr />

    ## Does PS Transcribe send audio to any cloud service?

    No. Speech recognition, voice-activity detection, and speaker diarization all run on your machine via FluidAudio. There are no API keys, no analytics endpoints, and no telemetry. You can verify with Little Snitch or the Network tab in Activity Monitor; the only outbound traffic is the Sparkle update check, which fetches an XML feed from GitHub and contains no session data.

    ## Which macOS versions are supported?

    macOS 26 or later. Earlier versions of macOS are not supported because the app uses APIs that ship in 26.

    ## Do I need Apple Silicon?

    Yes. PS Transcribe is built as an Apple Silicon binary; the FluidAudio models are tuned for the Apple Neural Engine. Running on Rosetta is not supported.

    ## Does it work on Intel Macs?

    No. Apple dropped Intel from macOS 26 at the OS level, and PS Transcribe requires macOS 26.

    ## Can I use it without an Obsidian vault?

    Yes. The "vault" is any folder you point at. Transcripts are plain markdown files with YAML frontmatter. Open them in any editor -- TextEdit, VS Code, iA Writer, anything that reads markdown.

    ## Is there a mobile version?

    No. PS Transcribe is macOS only. The Notion integration gives you read access from mobile if you route transcripts there, but recording happens on the Mac.

    ## What's the file format for saved transcripts?

    Markdown with YAML frontmatter. Speaker turns are rendered as blockquotes with a speaker label, one line per turn. The format is the same whether you use Obsidian, another editor, or just `cat` in the terminal.

    ## Can I edit transcripts after recording?

    Yes -- in whatever editor you use for the vault. PS Transcribe writes the file once on save and does not overwrite your edits. Inside the app, speaker names can be corrected with <Kbd>⌘</Kbd> <Kbd>E</Kbd> before you save; after save, edit the markdown directly.

    ## How accurate is speaker diarization?

    Good for 2–3 speakers in a clean recording; less good with more speakers, overlapping speech, or noisy audio. The diarizer auto-labels speakers as `Speaker 1`, `Speaker 2`, etc.; rename before saving and the frontmatter picks up the change.

    ## How do updates work?

    Sparkle polls the release feed on app launch. New versions appear as a quiet banner; clicking installs in place. Nothing is auto-installed without confirmation, and you can disable the check in **Preferences -- General**.

    # Troubleshooting

    <Lede>Common symptoms when PS Transcribe misbehaves and what to check first. Almost all of these come down to a macOS permission or a missing model download.</Lede>

    <hr />

    ## The app can't hear the other side of the call

    PS Transcribe needs **Screen &amp; System Audio Recording** permission, not just microphone access. The permission is granted at first launch, but if you denied it then or if macOS reset permissions after an update, re-grant it in *System Settings -- Privacy &amp; Security -- Screen &amp; System Audio Recording*. Restart PS Transcribe after toggling the permission.

    We do not record the screen. macOS bundles system-audio capture under the screen-recording API; there is no way to request one without the other.

    ## Speaker names are wrong

    The diarizer auto-labels speakers as `Speaker 1`, `Speaker 2`, and so on based on voice clustering. Click any transcript bubble and press <Kbd>⌘</Kbd> <Kbd>E</Kbd> to rename that speaker everywhere in the session. The frontmatter `participants` field picks up the new names at save time.

    If the diarizer merged two speakers or split one, there is no per-bubble correction yet -- rename to the best approximation and fix up in Obsidian after save. Tracked in the backlog.

    ## Models didn't download on first launch

    Parakeet-TDT and the diarization models are downloaded at first launch (roughly 400 MB total). If the download failed (no network, full disk, or interrupted), **Preferences -- Models -- Redownload** re-runs the fetch. Free up disk space first -- the installer is not resumable.

    <Note label="Disk space">
      The models unpack to roughly 800 MB on disk. Plan on 1.5 GB free before installing.
    </Note>

    ## Notion sync failed

    Three likely causes, in order:

    1. **Token is wrong or expired.** Regenerate the integration at [https://www.notion.so/my-integrations](https://www.notion.so/my-integrations) and paste the new token in **Preferences -- Integrations -- Notion**.
    2. **The integration isn't shared with your database.** Open the database in Notion, click the three-dot menu, choose **Connections -- Add connections**, and select your integration.
    3. **Property type mismatch.** PS Transcribe skips frontmatter fields whose Notion property exists but has the wrong type. Check the Notion activity feed to see which field was ignored.

    ## Transcript is slow to appear after I stopped

    Expected. When you press <Kbd>⌘</Kbd> <Kbd>.</Kbd>, the app runs voice-activity detection, final speech recognition, and speaker diarization on the full audio. This typically takes under a minute for a 30-minute call on Apple Silicon; longer for denser audio or more speakers. The live transcript you saw during recording was a preview pass, not the final one.

    ## Sparkle didn't show the update banner

    The Sparkle feed polls on launch, not while running. Quit and restart PS Transcribe; if a new version is out, the banner appears within a few seconds. To force a check, **PS Transcribe -- Check for Updates** in the menu bar. If no banner appears and you know a new version shipped, the appcast feed may be blocked by your network -- the feed URL is `https://github.com/cnewfeldt/ps-transcribe/releases.atom`.
