    NOTE: `_settings = State(initialValue: AppSettings())` is the required syntax for initializing @State in init(). You cannot use `self.settings = AppSettings()`.

    **Step 2 -- Add the migration function as a static method on PSTranscribeApp:**

    ```swift
    private static func migrateUserDefaultsIfNeeded() {
        let oldDomain = "io.gremble.tome"
        let sentinelKey = "hasMigratedFromTome"

        // Already migrated -- skip
        guard !UserDefaults.standard.bool(forKey: sentinelKey) else { return }

        // Open the old bundle ID's UserDefaults domain
        guard let oldDefaults = UserDefaults(suiteName: oldDomain) else { return }

        // Only migrate keys AppSettings actually reads
        let keysToMigrate = [
            "transcriptionLocale",
            "inputDeviceID",
            "vaultMeetingsPath",
            "vaultVoicePath",
            "hideFromScreenShare",
            "hasCompletedOnboarding",
        ]

        let newDefaults = UserDefaults.standard
        for key in keysToMigrate {
            if let value = oldDefaults.object(forKey: key) {
                newDefaults.set(value, forKey: key)
            }
        }

        // Mark migration complete
        newDefaults.set(true, forKey: sentinelKey)
        newDefaults.synchronize()

        // Delete old keys (D-04: clean break, no rollback)
        for key in keysToMigrate {
            oldDefaults.removeObject(forKey: key)
        }
        oldDefaults.synchronize()
    }
    ```

    CRITICAL per Research pitfalls:
    - Do NOT read old keys from `UserDefaults.standard` -- after bundle ID change, `.standard` is the NEW empty domain
    - Do NOT call `newDefaults.synchronize()` before all keys are written
    - Do NOT place migration in `onAppear` or `.task` -- it MUST be in `init()` before `@State var settings` initialization
    - The function MUST be `static` so it can be called before `self` is fully initialized
