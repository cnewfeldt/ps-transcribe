# Phase 3 -- UI Design Contract

> Visual and interaction contract for Phase 3: Session Management + Recording Naming.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
> This is a native macOS SwiftUI app -- no web component library or shadcn applies.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native SwiftUI) |
| Preset | not applicable |
| Component library | SwiftUI + AppKit (stdlib only) |
| Icon library | SF Symbols (system-provided) |
| Font | SF Pro (system font via `.system(size:weight:)`) |
| Color scheme | Dark only (`.preferredColorScheme(.dark)` enforced in ContentView) |

All design tokens are defined in `PSTranscribe/Sources/PSTranscribe/Views/TranscriptView.swift`
under `// MARK: - Design Tokens`. Every new view in Phase 3 MUST reference these tokens --
never hardcode hex values.

## Spacing Scale

Multiples of 4 only. Derived from existing usage across `ContentView.swift`, `TranscriptView.swift`,
and `ControlBar.swift`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label spacing within a row (e.g. missing badge next to name) |
| sm | 8px | Spacing between elements within a sidebar row (name-to-metadata gap) |
| md | 12px | Transcript bubble horizontal/vertical padding (established in TranscriptView) |
| lg | 16px | Horizontal insets for top bar, control bar, sidebar rows |
| xl | 24px | Vertical section gaps within settings form |
| 2xl | 32px | Reserved -- not required in this phase |
| 3xl | 48px | Empty state vertical centering padding |

Exceptions:
- Top bar height: 44px (established; matches macOS touch target minimum)
- Utterance bubble corner radius: 10px (established in TranscriptView)
- Sidebar column: min 180px, ideal 220px, max 300px (NavigationSplitView column width)
- Window minimum: 640px wide x 400px tall (sidebar 220px + detail 420px)

## Typography

All type uses SF Pro via `.font(.system(size:weight:))`. Existing sizes extracted from codebase:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 12px | regular (400) | 1.4 (system default) | Transcript utterance text, sidebar first-line preview |
| Label | 11px | medium (500) | 1.3 | Metadata (date, duration, source tag), settings form labels |
| Caption | 10px | bold (600) | 1.2 | Speaker label ("YOU", "THEM") with uppercase + tracking 0.8; missing badge tooltip |
| Brand | 14px | heavy (900) | 1.0 | "PS TRANSCRIBE" brand label in top bar (idle state) |

Name field (top bar active state): 14px, medium (500), plain style -- matches brand label size
to avoid layout shift when toggling between brand label and name field.

Two weights actively used in Phase 3 new UI:
- **regular (400):** body text, preview text, metadata values
- **medium (500):** name field, metadata labels, interactive labels

Heavy (900) is used only for the brand label -- not for new elements.

## Color

All tokens defined in `TranscriptView.swift` extension. Phase 3 adds NO new color tokens.

| Token | Hex | Role (60/30/10) | Usage |
|-------|-----|-----------------|-------|
| `Color.bg0` | #1A1818 | Dominant (60%) | App background, NavigationSplitView background |
| `Color.bg1` | #2E2B29 | Secondary (30%) | Sidebar row background (hover/selected), top bar glass, control bar glass, utterance bubbles |
| `Color.bg2` | #242120 | Secondary (30%) | Sidebar column background (unselected surface) |
| `Color.fg1` | #F0EDE8 | Text primary | Recording name, sidebar entry title, active state text |
| `Color.fg2` | #8A8480 | Text secondary | Metadata (date, duration), placeholder text, idle status |
| `Color.fg3` | #5C5854 | Text tertiary | Caption text, disabled/empty state secondary copy |
| `Color.accent1` | #C4A0FF | Accent (10%) | Active recording indicator (pulsing dot), Obsidian icon button when active, inline save confirmation, selected sidebar row left border stripe |
| `Color.recordRed` | #E85B5B | Semantic: error | Missing file badge indicator only |
| `Color.accent2` | #9678BF | Accent dimmer | Obsidian icon button when vault not configured (disabled state) |

**Accent reserved for (explicit list):**
- Pulsing dot in top bar during active recording
- Left-edge stripe on selected sidebar entry (3px wide)
- Obsidian icon button (enabled state only)
- Inline save confirmation checkmark icon
- Sidebar entry "active recording" ring or badge

**Destructive color (`Color.recordRed`):**
- Missing file badge icon (`.orange` is an alternative -- use `recordRed` for consistency with existing error pattern)
- Never used for CTA buttons, never used decoratively

**Do not use accent1 for:**
- Hover states on non-selected sidebar rows
- General button tints
- Background fills (opacity > 0.15 on any surface)

## Layout Architecture

This phase introduces a structural change from single-column VStack to NavigationSplitView.

### Window

```
┌──────────────────────────────────────────────────────────────┐
│ [Top Bar: brand label OR name field]                [status] │
├───────────────────┬──────────────────────────────────────────┤
│  Sidebar          │  Detail                                  │
│  (min 180px)      │  (flex)                                  │
│                   │                                          │
│  [Library list]   │  [TranscriptView]                        │
│  or [Empty state] │  or [Empty detail state]                 │
│                   │                                          │
├───────────────────┴──────────────────────────────────────────┤
│ [ControlBar]                                                  │
└──────────────────────────────────────────────────────────────┘
```

- `NavigationSplitView` with `.balanced` style
- `.frame(minWidth: 640, minHeight: 400)` on root
- Sidebar column: `.navigationSplitViewColumnWidth(min: 180, ideal: 220, max: 300)`
- Top bar spans full width (outside NavigationSplitView, in outer VStack)
- ControlBar spans full width (outside NavigationSplitView, in outer VStack)
- Source: RESEARCH.md Pattern 3, CONTEXT.md D-01

### Sidebar Column

- Background: `Color.bg2` (unselected surface)
- Selected row: `Color.bg1` with `Color.accent1` left-edge stripe (3px)
- Row height: 72px (fits name + 2 lines metadata, matches macOS standard list row height range)
- No separator lines between rows -- use background contrast only
- Sorted: most recent first (CONTEXT.md D-03)

### Detail Column

Three states:

1. **Active recording:** `RecordingNameField` (top sub-bar) + `TranscriptView` (live)
2. **Past transcript loaded:** Loaded transcript header + `TranscriptView` (read-only, no volatile text)
3. **No selection:** Centered empty detail message

## Interaction Contracts

### Top Bar -- Dual Mode (NAME-01, NAME-02)

**Idle mode (no active session):**
- Shows: `Text("PS TRANSCRIBE")` -- 14px heavy, tracking 3, `Color.fg1`
- Right side: "Ready" in 11px medium `Color.fg2` + static circle dot `Color.fg2` opacity 0.5
- Not interactive

**Active/post-session mode:**
- Shows: `TextField("Name this recording", text: $sessionName)` -- 14px medium, `.textFieldStyle(.plain)`, `Color.fg1`
- Placeholder: "Name this recording" in `Color.fg2`
- Right side: timer in 11px medium `Color.fg1` + `PulsingDot` (recording) or "Done" (stopped)
- Editable before, during, and after recording (CONTEXT.md D-05)
- onChange debounced 500ms before calling `TranscriptLogger.setName()` -- RESEARCH.md Pitfall 4
- On resign focus: commit name immediately (no debounce delay)

Transition: no animation -- straightforward conditional rendering avoids layout shift because both
states use 14px text at the same frame height (44px).

### Sidebar Entry Row (SESS-02, NAME-03)

Each row (72px height, 16px horizontal padding) contains:

```
[type icon]  [recording name]                    [obsidian icon]
             [date]  [duration]  [source tag]    [missing badge]
             [first-line preview text]
```

**Type icon:** SF Symbol
- `.callCapture`: `phone.fill` 14pt `Color.fg2`
- `.voiceMemo`: `mic.fill` 14pt `Color.fg2`
- Incomplete/recovered session: `exclamationmark.circle` 14pt `Color.fg2`

**Recording name:**
- Display: `Text(entry.displayName)` -- 13px medium `Color.fg1` (13px not in main type scale --
  justified exception for sidebar list density; equivalent to Notes.app sidebar)
- Click-to-edit: tap name text -> TextField appears, `onSubmit` commits, `onExitCommand` cancels
- TextField placeholder: entry's current displayName (so user sees what they're editing)
- Commit triggers: Enter key OR focus loss
- On commit: `LibraryStore.updateEntry()` + `TranscriptLogger.atomicRewrite()` (RESEARCH.md Pattern 6)

**Metadata line:** 11px regular `Color.fg2`
- Format: "Apr 2, 2026 · 14m 32s · Teams"
- Date: `DateFormatter` with "MMM d, yyyy"
- Duration: formatted as "Xh Xm Xs" omitting leading zero units (e.g. "32s", "14m 32s", "1h 2m")
- Source tag: detected app name from `conferencingBundleIDs` or "Voice Memo"

**First-line preview:** 11px regular `Color.fg3`, single line, truncated with `.truncationMode(.tail)`
- Shows first non-empty utterance text from transcript
- Stored in `LibraryEntry` at session stop (not read from file on render)
- Empty transcript: show "No transcript" in `Color.fg3`

**Missing file badge:**
- SF Symbol `exclamationmark.triangle.fill` 10pt, `Color.recordRed`
- `.help("File has been moved or deleted")` for hover tooltip
- Shown when `FileManager.default.fileExists(atPath: entry.filePath)` returns false
- Source: REQUIREMENTS.md SESS-05, RESEARCH.md code example

**Obsidian icon button:**
- SF Symbol `link` 12pt (no official Obsidian SF symbol -- use `link` as proxy)
- Enabled: `Color.accent1`, opens `obsidian://` URL via `NSWorkspace.shared.open()`
- Disabled (vault name not configured): hidden entirely (CONTEXT.md D-12)
- `.help("Open in Obsidian")` tooltip

**Selected row state:**
- Background: `Color.bg1.opacity(0.85)` (slightly stronger than the 0.7 used on utterance bubbles)
- Left stripe: 3px `Color.accent1` (same pattern as utterance bubble speaker indicator)
- `.clipShape(RoundedRectangle(cornerRadius: 6))` on the background -- not on the full row

**Hover state:** `Color.bg1.opacity(0.4)` via `.onHover` -- no animation, instant toggle

### Session Stop -> Library (SESS-07, D-08)

After `stopSession()`:
1. New `LibraryEntry` added to `LibraryStore` (appears at top of sidebar)
2. New entry auto-selected in sidebar
3. Transcript stays visible in detail column (no clear)
4. Inline save confirmation shown: a single line in top bar replacing the timer --
   `"Saved"` with a checkmark icon in `Color.accent1`, fading out after 2 seconds
5. No full-screen banner -- the existing `saveBanner` is retired in this phase

The "Show in Finder" action moves from the save banner to the library entry context menu
(right-click -> "Show in Finder").

### Session Start (D-09)

Starting a new recording while a past transcript is selected:
- No confirmation dialog
- Detail column switches immediately to live recording view
- New entry appears at top of sidebar with `isFinalized: false`
- Past transcript entry remains in sidebar and is accessible via tap

### Library Empty State (SESS-01, D-04)

Shown when `LibraryStore.entries.isEmpty`:

```
[waveform.circle icon -- 28pt, Color.fg3]
"No recordings yet"                          [13px bold, Color.fg2]
"Start a call capture or voice memo          [11px regular, Color.fg3, centered]
 to begin transcribing."
```

Vertically centered in sidebar column. No action button -- guidance points to ControlBar controls
which are always visible. Source: CONTEXT.md D-04.

### Detail Empty State (no selection, library has entries)

```
[doc.text icon -- 24pt, Color.fg3]
"Select a recording"                         [12px bold, Color.fg2]
"Choose a session from the sidebar"          [11px regular, Color.fg3]
```

Vertically centered in detail column.

### Incomplete Session Entries (STAB-01 integration)

Sessions recovered from crash checkpoints appear in library with:
- Name: date-based fallback (not "Recovered" -- that's internal source tracking)
- Type icon: `exclamationmark.circle` 14pt `Color.fg2` instead of phone/mic icon
- Source tag: "Recovered" (in `Color.fg2`)
- `isFinalized: false` -- no Obsidian button shown
- If file exists: tapping loads transcript; if missing: shows missing badge

### Obsidian Vault Name Setting (D-11)

In `SettingsView.swift`, new row in "Output Folders" section:

```
Section("Output Folders") {
    // ... existing vault path rows ...

    HStack {
        VStack(alignment: .leading, spacing: 2) {
            Text("Obsidian Vault Name")     [12px medium]
            Text("Used to open transcripts in Obsidian")  [11px, tertiary]
        }
        Spacer()
        TextField("e.g. MyVault", text: $settings.obsidianVaultName)
            .frame(width: 120)
            .multilineTextAlignment(.trailing)
    }
}
```

No "Choose..." button -- user types the vault name. One-time setup. No auto-detection.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (save action) | "Saved" (inline confirmation, not a button) |
| Top bar name placeholder | "Name this recording" |
| Library empty state heading | "No recordings yet" |
| Library empty state body | "Start a call capture or voice memo to begin transcribing." |
| Detail empty state heading | "Select a recording" |
| Detail empty state body | "Choose a session from the sidebar" |
| Missing file tooltip | "File has been moved or deleted" |
| Obsidian button tooltip | "Open in Obsidian" |
| Show in Finder (context menu) | "Show in Finder" |
| Inline rename placeholder | [entry's current displayName -- dynamic] |
| Unnamed recording fallback (call) | "Call Recording -- Apr 2, 2026" |
| Unnamed recording fallback (voice) | "Voice Memo -- Apr 2, 2026" |
| Error: transcript load failure | "Couldn't load transcript. The file may be corrupted." |
| Obsidian vault name label | "Obsidian Vault Name" |
| Obsidian vault name helper | "Used to open transcripts in Obsidian" |
| Obsidian vault name placeholder | "e.g. MyVault" |

**Destructive actions in this phase:** None. Renaming is non-destructive (original file replaced
atomically). No delete session in this phase.

**Tone:** Minimal, functional, lowercase where possible except proper nouns. Match existing
"Start a call capture or voice memo" pattern already in the codebase.

## Animation Contract

| Interaction | Animation | Duration | Value |
|-------------|-----------|----------|-------|
| New sidebar entry appears | `.animation(.easeOut(duration: 0.2))` | 200ms | Matches existing utterance scroll animation |
| Sidebar row hover | None -- instant color change | -- | Matches macOS native list behavior |
| Inline save confirmation fade-out | `.opacity` transition | 500ms fade after 2s delay | `withAnimation(.easeOut(duration: 0.5))` |
| Pulsing dot (active recording) | Established -- reuse `PulsingDot` | 0.9s ease-in-out repeat | Already in ControlBar.swift |
| Sidebar entry name -> TextField | None -- instant swap | -- | Avoid animation on state change to input |

No new animation patterns. Reuse existing `.easeOut(duration: 0.2)` as the standard duration
for any state transitions that require animation.
